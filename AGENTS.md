# Project Rules

## 工作流

- 后续计划和说明优先用中文。
- 读取含中文的项目文件时显式使用 UTF-8。
- 完成任何需求开发或 bug 修复后提交 git；PowerShell 中分步执行命令，不使用 `&&`。
- 若进行了 UI 改动，必须用本地页面做一次实际浏览器检查。
- 将用户纠正过的项目规则继续记录到 `AGENTS.md`，但要合并归类，避免流水账。
- 新建工作树后，`apply_patch` 必须确认目标路径在新工作树内；完成后询问是否合并回主分支并删除工作树和分支。
- 使用 `rg` 搜索包含括号等正则特殊字符的内容时，优先用固定字符串或多次简单搜索。

## UI 与浏览器交互

- 禁用原生 `alert`、`confirm`、`prompt`；所有确认、错误、权限提示都使用应用内弹窗或就近状态。
- 不暴露原生浏览器滚动条；滚动区域使用项目自定义滚动条样式。
- 不使用 `input[type=file][webkitdirectory]` 作为目录选择 fallback；目录选择使用 File System Access API，不支持时给应用内提示。
- 修复 UI 溢出时同时检查文本横向约束、换行策略、行高、高度和实际裁切方向；用户明确要求调高统计栏时，优先增加容器及子项高度。
- 移动端按钮同排问题要同时检查全局按钮宽度和 flex 子项收缩规则。
- 浅色/深色适配必须同时检查 `.app-shell.theme-light` 与 `:root[data-theme="light"]`。
- 标签输入触发大模型 API 查询时，在按钮或就近控件显示 loading 状态和动态图标。

## 数据身份与持久化

- `label`、Tag 都是用户可重复的显示元数据，不能作为唯一标识；持久化关联、合并或相似关系使用归一化 key、`videoId` 或全局 id。
- 全局媒体库下 `video.id` 必须包含 `mediaRootId`；旧数据迁移用旧相对路径 id 映射到全局 id，不能假设单库内唯一。
- 浏览器添加的媒体根目录只有目录句柄和相对路径，不等于服务端本机绝对路径；补本机绝对路径时写入 `localPath`，不要覆盖 `path`。

## 媒体库与扫描

- 依赖 `ffmpeg`/`ffprobe` 前必须确认 `source !== "browser"` 或存在已验证的 `localPath`；不要让入口可点击后才报错。
- 不要在首页或全局扫描阶段批量对每个视频运行 `ffprobe`/`ffmpeg`；兼容性探测按需触发、缓存结果或限制并发。
- 全局媒体库服务端扫描必须复用小文件和特殊命名视频过滤规则，如 `theme_video`、`trailer`。
- 全局媒体库和写真集扫描接口必须合并并发请求，避免 React StrictMode 或重复操作同时遍历大目录。
- Vite dev server 必须忽略 `.local-web-player-data`、`dist` 等本地生成目录监听，避免 `EMFILE`。
- 片库搜索是当前媒体模式内的全局搜索：追番只搜 Anime，特殊模式只搜特殊库，全部模式才搜全部库；不要用继续观看卡片当主要候选范围。
- AI 搜索没有有效 `matchIds` 时，不展示模型对某个候选的自由文本解释。
- 无剧透回顾入口只在追番模式显示；追番模式下即使没有未看完进度或 `primaryResumeCard`，也显示空状态并禁用生成按钮。

## 写真集

- 写真集目录首次必须由用户手动选择；之后默认复用已保存的单一写真集目录句柄，不主动扫描媒体库或本机磁盘。
- 写真集目录选择使用 `showDirectoryPicker({ mode: "readwrite" })`；若已保存句柄没有写权限，自动清除句柄和扫描缓存并回到空状态。
- 写真集界面顶部不显示全局“新增媒体库”，工具栏不重复放“选择文件夹”；保留空状态里的“选择写真集文件夹”入口。
- 写真集数量或单本图片过多时使用分页、窗口化或同等策略，避免一次性渲染全部卡片或缩略图。
- 浏览器目录扫描不要为所有图片提前创建 blob URL；只为当前分页封面、当前阅读图片和可见缩略图按需创建，并在翻页或切换视图时回收。
- 阅读页顶部不显示写真集名称/路径；收藏、标记已读、重读按钮靠右并保持同一行。
- 低分辨率图片等比放大时，图片元素本身占满舞台尺寸，再用 `object-fit: contain` 保持比例。
- 删除浏览器目录里的视频或写真图片时，只用 `queryPermission` 检查权限，不调用 `requestPermission({ mode: "readwrite" })`；无权限用应用内提示中止。
- 写真集图片删除必须用应用内确认弹窗；缓存图片缺少 `parentDirectory` 时，通过已保存根目录和图片相对路径重新解析父目录。
- 删除图片后仅更新当前写真集缓存条目：移除图片、更新数量/大小/更新时间，必要时移除空写真集；不要为单张删除重扫整个写真目录。
- 删除确认后反查文件是否仍存在，失败原因显示在当前弹窗内。

## React 与异步状态

- React state updater 必须保持纯函数；不要在 updater 内修改 ref 或其他外部状态。
- 设置 `isScanning` 的扫描函数必须在函数内部用 `finally` 复位，覆盖权限拒绝、用户取消和提前返回。
- 使用 runId 或取消标记丢弃旧异步结果的 effect，不能把 `loading` 当成可跳过的终态；新一轮 effect 必须重新发起请求或接管旧请求。
