# Project Rules

- Do not use native browser dialogs such as `alert`, `confirm`, or `prompt`; use app-styled modal UI instead.
- Do not expose native browser scrollbars; style scrollable areas with the app's custom scrollbar treatment.
- Media library names (`label`) are display names and may be duplicated; never use `label` as a uniqueness constraint.
- 将你犯过的错、我提过的提醒记录下来，写进 `AGENTS.md`。
- 后续输出计划需要用中文展示。
- 浏览器添加的媒体根目录只有目录句柄和相对路径，不等于服务端可访问的本机绝对路径；依赖 ffmpeg/ffprobe 的功能必须先确认 `source !== "browser"` 或存在已验证的 `localPath`，否则不要把入口做成可点击后才报错。
- 给浏览器添加的媒体库补本机绝对路径时，保存在 `localPath`；不要覆盖 `path`，因为 `path` 是浏览器目录标识。
- Tag 是用户输入的显示元数据，不得作为唯一标识；涉及合并、相似关系或持久化关联时使用归一化 key 或 `videoId`。
- 读取包含中文的项目文件时显式使用 UTF-8，避免把 PowerShell 默认编码导致的乱码误判为源码损坏。
- React state updater 必须保持纯函数；不要在 updater 内修改 ref 或其他外部状态，否则 StrictMode/并发渲染可能让自动选择等逻辑计算两次后回退。
- 全局媒体库下 `video.id` 必须包含 `mediaRootId`；跨媒体根迁移旧数据时用旧相对路径 id 映射到全局 id，不能再假设单库内唯一。
- 全局媒体库服务端扫描也必须复用小文件和特殊命名视频过滤（如 `theme_video`、`trailer`），不要只在浏览器目录扫描路径过滤。
- 修复 UI 溢出问题时不能只调整高度/间距或只按横向溢出推断；必须检查文本横向约束、换行策略、行高和实际裁切方向，避免“总大小/文件数/缓存种类/最近更新”等标签继续溢出或被底部裁切。
- 用户明确指出统计栏被裁切并要求“直接把高度调高”时，优先增加容器和子项高度，不要继续用换行/省略策略绕问题。
- 写真集目录首次必须由用户手动选择；之后默认复用已保存的单一写真集目录句柄，不要主动扫描已配置媒体库或本机磁盘。
- 不要使用 `input[type=file][webkitdirectory]` 作为文件夹选择 fallback；它会触发“上传 N 个文件到此网站”的浏览器原生确认框，破坏应用内弹窗体验。需要选择文件夹时使用 File System Access API，浏览器不支持时给应用内提示。
- 写真集界面顶部不要显示全局“新增媒体库”，工具栏也不要重复放“选择文件夹”；保留空状态里的“选择写真集文件夹”入口即可。
- 写真集数量或单本图片过多时，不要一次性渲染全部卡片或缩略图；使用分页、窗口化或同等策略控制 DOM 和图片请求量。
- 写真集浏览器目录扫描不要为所有图片提前创建 blob URL；只为当前分页封面、当前阅读图片和可见缩略图按需创建，并在翻页或切换视图时回收。
- 写真集阅读页顶部不要显示写真集名称/路径信息；收藏、标记已读、重读操作按钮靠右并保持同一行。
- 修复写真集阅读页顶部按钮同排问题时，必须同时检查移动端全局按钮宽度（如 `.secondary-button { width: 100%; }`）和 flex 子项收缩规则，避免右侧按钮组仍被撑成竖排。
- 做浅色/深色主题适配时，同时检查 `.app-shell.theme-light` 和 `:root[data-theme="light"]` 两套选择器，避免只覆盖其中一套导致弹窗内标签等组件文字发白。
- 标签输入触发大模型 API 查询时，必须在按钮或就近控件显示 loading 状态和动态图标，避免用户误以为前端卡死。
- 不要在首页/全局媒体库扫描阶段对每个视频批量运行 `ffprobe`/`ffmpeg`；这会让大媒体库长时间显示扫描中并造成巨大磁盘读写。媒体兼容性探测必须按需触发、缓存结果或限制并发。
- Vite dev server 必须忽略 `.local-web-player-data`、`dist` 等本地生成目录的文件监听，避免缓存文件过多导致 `EMFILE: too many open files`。
- 全局媒体库/写真集扫描接口必须合并同时发生的请求，避免 React StrictMode 或重复操作并发遍历多个大媒体库。
- 无剧透回顾入口只在追番模式显示，但追番模式下不能因为没有未看完进度或没有 `primaryResumeCard` 就整块隐藏；应显示空状态并禁用生成按钮。
- 每完成一个需求开发或 bug 修复后，都要将此次相关修改提交到 git。
- 当前 PowerShell 环境不支持 `&&` 作为命令连接符；需要分步执行 git add/commit 等命令，避免提交流程被解析错误打断。
- 使用 `rg` 搜索包含括号的模式时，优先用简单多次搜索或固定字符串选项，避免手写未闭合正则导致搜索失败。
